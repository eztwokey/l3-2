# Shortener — сервис сокращения URL с аналитикой

## Описание

Сервис генерирует короткие ссылки, перенаправляет пользователей на оригинальные URL и собирает аналитику переходов: количество кликов, User-Agent, IP-адрес, агрегация по дням и месяцам.

## Технологии

- **Go 1.25** — язык разработки
- **Gin** — HTTP-фреймворк
- **PostgreSQL 16** — основное хранилище (ссылки, клики, аналитика)
- **Redis** — кэширование популярных ссылок для быстрых редиректов
- **wbf** — внутренняя библиотека (dbpg, redis, logger, config)
- **Docker Compose** — оркестрация инфраструктуры

## Архитектура

Слоёная архитектура с разделением ответственности:
```
HTTP-запрос → API (хендлеры) → Logic (бизнес-логика) → Storage (PostgreSQL + Redis)
```

- **API** — принимает HTTP-запросы, извлекает параметры, возвращает ответы
- **Logic** — валидация, генерация кодов, координация между storage-методами
- **Storage** — SQL-запросы к PostgreSQL, кэширование в Redis
- **Interfaces** — контракты для storage (позволяют подменить реализацию в тестах)

## Схема БД

**links** — сокращённые ссылки:
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | первичный ключ |
| short_code | VARCHAR(20) UNIQUE | короткий код |
| original_url | TEXT | оригинальный URL |
| created_at | TIMESTAMP | дата создания |

**clicks** — переходы по ссылкам:
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | первичный ключ |
| link_id | INT (FK → links.id) | ссылка на links |
| user_agent | TEXT | браузер/устройство |
| ip_address | VARCHAR(45) | IP посетителя |
| clicked_at | TIMESTAMP | дата перехода |

## API

| Метод | Путь | Описание |
|-------|------|----------|
| POST | /shorten | Создать короткую ссылку |
| GET | /s/:code | Перейти по короткой ссылке (302 redirect) |
| GET | /analytics/:code | Аналитика по ссылке |

## Кэширование

При создании ссылки и первом переходе original_url кэшируется в Redis (ключ `short:{code}`, TTL 24 часа). При редиректе сначала проверяется Redis, при промахе — PostgreSQL с последующим кэшированием.

## Запуск
```bash
# 1. Поднять PostgreSQL и Redis
make start

# 2. Скопировать и заполнить конфиг
cp config/config.yaml.example config/config.yaml

# 3. Запустить (миграции выполнятся автоматически)
make run
```
